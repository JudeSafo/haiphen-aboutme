Installation

Everything you need to install and run Haiphen across every platform. Choose the CLI for headless workflows, the Desktop app for a graphical interface, or the Mobile app for monitoring on the go.

CLI (Command Line Interface)

The Haiphen CLI is a single binary built with Go. It includes a local gateway, GitHub OAuth login, entitlement checks, and service health monitoring.

macOS via Homebrew

brew tap haiphenAI/tap
brew install haiphen

Linux via curl installer

curl -fsSL https://haiphen.io/install.sh | bash

The installer detects your architecture (amd64 / arm64), downloads the latest release, and places the binary in /usr/local/bin.

Windows via GitHub Releases

# Download the latest .exe from GitHub Releases:
# https://github.com/haiphenAI/haiphen-aboutme/releases

# Or use Scoop:
scoop bucket add haiphen https://github.com/haiphenAI/scoop-bucket
scoop install haiphen

Build from source (Go 1.22+)

git clone https://github.com/haiphenAI/haiphen-aboutme.git
cd haiphen-cli
go build -o haiphen ./cmd/haiphen/main.go
./haiphen --help

Docker

# Quick check
docker run --rm -it ghcr.io/haiphenai/haiphen:latest --help

# Run the local gateway on port 8787
docker run --rm -p 8787:8787 ghcr.io/haiphenai/haiphen:latest serve

Verify installation

haiphen --version
haiphen status

haiphen --version prints the release tag and build date. haiphen status checks your authentication state and current plan entitlement.

Quick Start Guide

After installing the CLI, get up and running in five commands:

# 1. Authenticate with GitHub (opens browser)
haiphen login

# 2. Verify your entitlement and plan tier
haiphen status

# 3. Start the local gateway on :8787
haiphen serve

# 4. Check the health of all Haiphen services
haiphen services

# 5. Open the local dashboard
open http://localhost:8787

CLI Rate Limiting Behavior

The CLI enforces rate limits at two layers to keep your workflow smooth and protect shared infrastructure.

  • Local sliding window pre-filter — the CLI tracks request counts in memory and rejects calls that would exceed your tier, preventing unnecessary round trips to the server.
  • Server-side QuotaDO enforcement — a Durable Object tracks daily consumption per account with the following limits:
PlanDaily Limit
Free200
Pro10,000
Enterprise50,000

When you exceed your daily quota the CLI displays a message with an upgrade link to haiphen.io/#services.

Configuration

All CLI configuration is stored at ~/.config/haiphen/.

Profile management

Haiphen supports multiple profiles for switching between accounts or environments. Use the --profile flag with any command: 

# Login with a named profile
haiphen --profile work login

# Check status of a specific profile
haiphen --profile work status

# Start the gateway using a profile
haiphen --profile work serve

Session tokens are stored in profile-specific JSON files at ~/.config/haiphen/session.<profile>.json. The default profile is default.

Troubleshooting

SymptomCauseSolution
haiphen login does not open a browser No default browser configured, or running inside SSH / container Copy the URL printed in the terminal and paste it into any browser manually
Unauthorized after a successful login JWT has expired (tokens are short-lived) Run haiphen login again to obtain a fresh token
Rate limit exceeded / 429 response Daily quota consumed for your plan tier Check remaining quota with haiphen status; upgrade at haiphen.io/#services
haiphen serve port already in use Another process is bound to port 8787 Stop the conflicting process or set an alternate port: haiphen serve --port 9090
haiphen services shows unhealthy endpoints Transient network issue or planned maintenance Retry after a few minutes; check status.haiphen.io for incident updates
Docker container exits immediately Missing command argument Provide a subcommand: docker run --rm -it ghcr.io/haiphenai/haiphen:latest serve
Build from source fails with Go version error Go version older than 1.22 Upgrade Go to 1.22 or later: go install golang.org/dl/go1.22.0@latest
